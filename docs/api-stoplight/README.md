# API DOC Guide

## Goal
- Manage API docs directly on the site.
- Create folders and API DOC pages first, then fill template fields.
- Show final doc in Spotlight-style preview for FE/BE interface review.

## UI Entry
- API DOC editor: `http://localhost:3000/api-docs`

## How To Work
1. Create folder in left sidebar.
2. Create API DOC page inside folder (or Ungrouped).
3. Fill template fields:
   - title, method, path, status, auth
   - purpose
   - path/query/header
   - request/response examples
   - frontend handling notes
4. Verify Spotlight preview panel.
5. Move status to `approved` after FE/BE agreement.
