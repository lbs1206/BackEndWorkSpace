# API Review Checklist

- [ ] Status is correct (`draft/review/approved/deprecated`)
- [ ] Endpoint basics are filled (title, method, path, auth)
- [ ] Purpose is clear for frontend feature context
- [ ] Path/query/header sections are documented
- [ ] Request body example is present
- [ ] Success response example is present
- [ ] Error response example is present (`code/message/traceId`)
- [ ] Field naming style is consistent (`camelCase` or `snake_case`)
- [ ] Frontend handling notes are included
