# ERD Policy

## 목적

ERD 변경으로 인한 스키마 충돌, 누락, 배포 리스크를 줄이기 위해 사전 합의 기준을 정의한다.

## 변경 단위

1. 엔티티 추가/삭제
2. 컬럼 추가/삭제/타입 변경
3. PK/FK/UNIQUE/INDEX 변경
4. 정규화/비정규화 구조 변경

## 필수 작성 항목

1. 변경 배경
2. 대상 테이블/컬럼
3. 제약조건 영향
4. 데이터 마이그레이션 방법
5. 롤백 방법
6. API 영향 범위

## 명명 규칙

1. 테이블: `snake_case` 복수형
2. 컬럼: `snake_case`
3. FK: `<target_table>_id`
4. 인덱스: `idx_<table>_<column>`
5. 유니크: `uk_<table>_<column>`

## 검토 체크리스트

1. nullable/기본값 정의가 명확한가
2. FK 삭제/수정 시 참조 무결성 문제가 없는가
3. 조회 패턴에 맞는 인덱스가 있는가
4. 대용량 테이블 변경 시 lock/time 영향 분석이 있는가
5. 하위 호환(Backward Compatibility)을 유지하는가

## 승인 기준

1. 백엔드 1인 + DB 담당 1인 승인
2. 마이그레이션 스크립트 검증 완료
3. 롤백 시나리오 리허설 또는 문서화 완료
